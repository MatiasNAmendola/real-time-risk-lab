# Enterprise Go Layout Reference

Referencia conceptual para adaptar la separación enterprise típica de Go a módulos Java con Clean Architecture.

## Idea

Separar explícitamente `domain`, `application` e `infrastructure` para que las dependencias apunten hacia adentro y los adapters queden fuera del dominio.
